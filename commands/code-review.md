Проведи полный code review текущего проекта по чеклисту из @~/.claude/coding-pitfalls.md

## Что проверить

### 1. Безопасность
- Есть ли секреты/пароли/токены захардкоженные в коде?
- Есть ли SQL без параметризации (string concatenation)?
- Есть ли innerHTML / dangerouslySetInnerHTML с пользовательским вводом?
- Есть ли auth endpoints без rate limiting?
- Есть ли .env или credentials в git?

### 2. Обработка ошибок
- Есть ли проглоченные ошибки (`_ = err`, `except: pass`, пустой `catch {}`)?
- Есть ли функции которые возвращают дефолтные значения вместо ошибок?
- Все ли ошибки залогированы или возвращены вызывающему коду?

### 3. Архитектура
- Есть ли God Objects (struct/class с > 5 зависимостями, > 300 строк)?
- SQL в handler-ах вместо repository?
- Бизнес-логика в handler-ах вместо service?
- Дублирование кода (одинаковая логика в > 2 файлах)?

### 4. Качество кода
- Магические числа без именованных констант?
- Функции > 30 строк?
- Вложенность > 3 уровней?
- Плохие имена переменных (x, tmp, data, flag)?
- Copy-paste код?

### 5. Тесты
- Есть ли тесты вообще?
- Покрыты ли edge cases (null, пустой массив, 0)?
- Тесты на реализацию вместо поведения?
- Flaky tests (time.Sleep, зависимость от порядка)?

### 6. Go-специфичное (если Go проект)
- Запусти `go vet ./...`
- Проверь race conditions: есть ли shared state без sync?
- Есть ли горутины без proper cleanup?

### 7. Dart/Flutter-специфичное (если Flutter проект)
- Запусти `dart analyze`
- Есть ли BuildContext через async gap?
- Dispose вызывается для всех контроллеров?

## Формат вывода

Для каждой найденной проблемы:
```
[КАТЕГОРИЯ] файл:строка — описание проблемы
  Что не так: ...
  Как исправить: ...
```

В конце — общая оценка:
- Критичные проблемы (блокируют мерж)
- Предупреждения (стоит исправить)
- Рекомендации (улучшения)
