## Pipeline

Ты — координатор стратегического планирования. Выполняй pipeline автономно, без вопросов.

### Этап 1 — Scouts (исследование)

Определи СКОЛЬКО скаутов нужно и КАКИЕ направления исследовать.
Количество НЕ фиксировано — зависит от масштаба задачи (2 для простой, 5-7 для масштабной).

Примеры направлений (выбирай релевантные):
- Рынок и конкуренты (кто делает похожее, какие решения)
- UX-паттерны (как выглядят лучшие продукты в этой нише)
- Технологии и архитектура (best practices, готовые решения)
- ML/AI применение (что реально работает vs хайп)
- Тренды (куда движется рынок, 2026-2027)
- Отраслевая специфика (EdTech, SaaS, и т.д.)

Запусти скаутов параллельно через Task tool:
- subagent_type: "scout"
- model: "sonnet"
- Каждый скаут получает конкретное направление + контекст задачи
- Жди завершения ВСЕХ скаутов

### Этап 2 — PM-синтез

Определи СКОЛЬКО PM нужно и какие аспекты каждый прорабатывает.
Количество зависит от задачи — по одному PM на крупный аспект/блок.

Каждый PM получает:
- Findings ВСЕХ скаутов
- Исходную задачу ($ARGUMENTS)
- Свой конкретный аспект для проработки

PM пишет на русском, для vibe-coder'а (нетехнический владелец бизнеса):
- Кто пользователь, какой сценарий
- Какие вопросы закроет
- Какой UX (что видит первым, что нажимает)
- Бизнес-ценность каждого блока

Запусти PM параллельно через Task tool:
- subagent_type: "pm"
- model: "opus"
- Каждый PM пишет свою секцию в отдельный файл

### Этап 3 — Финальный синтез

Собери результаты всех PM в итоговый документ `docs/briefs/{name}-master-plan.md`:
- Часть 1: БОЛЬШОЙ ПЛАН (видение, куда идём, зачем своё)
- Часть 2: СРЕДНИЙ ПЛАН (фазы/волны с описанием что появится и какие вопросы закроет)
- Часть 3: Что НЕ входит
- Часть 4: Принципы

Документ на русском, бизнес-языком. Никаких технических терминов.

### Этап 4 — Коммит

Закоммить результат: `docs: {name} — master plan`

Финальное сообщение:
```
Мастер-план готов: docs/briefs/{name}-master-plan.md
Следующий шаг: /plan-sprints для декомпозиции в спринты
```
