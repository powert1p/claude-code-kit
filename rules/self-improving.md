# Self-Improving

## Правило
После 3+ неудачных попыток исправить ошибку — абстрагируй урок и запиши.

## Формат урока
- ALWAYS/NEVER директива + конкретная команда/паттерн
- Пример: "NEVER use $1 twice in asyncpg query — each param needs unique $N"

## Куда писать
- Универсальный урок → MEMORY.md (секция "Lessons Learned")
- Специфичный для домена → соответствующий rule-файл (sql.md, backend.md, frontend.md)

## Anti-bloat
- Макс 50 уроков в MEMORY.md. Перед добавлением проверь дубликат через Read
- Если лимит достигнут — объедини похожие или удали устаревшие
- Формат: 1 строка на урок. Без многословных объяснений
